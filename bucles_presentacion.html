¿Qué es un bucle?

Un bucle permite repetir un bloque de código múltiples veces sin tener que escribirlo una y otra vez.

🔁

Sin bucle

Tienes que escribir la misma instrucción 10 veces manualmente. Repetitivo, largo y propenso a errores.

⚡

Con bucle

Le dices al programa: "repite esto X veces" o "repite esto mientras se cumpla una condición". Simple y poderoso.

Bucle for

Úsalo cuando sabes exactamente cuántas veces quieres repetir algo.

for (int i = 1; i <= 5; i++) {
    cout << "Iteración número: " << i << endl;
}
INICIO
int i = 1

Se ejecuta una sola vez al comenzar. Inicializa el contador.

CONDICIÓN
i <= 5

Se evalúa antes de cada repetición. Si es falsa, el bucle termina.

INCREMENTO
i++

Se ejecuta al final de cada iteración. Avanza el contador.

Bucle while

Úsalo cuando no sabes cuántas veces se va a repetir — depende de una condición externa.

int numero = 0;

while (numero != 5) {
    cout << "Ingresa el número 5: ";
    cin >> numero;
}
CONDICIÓN
numero != 5

Se evalúa antes de cada vuelta. Mientras sea verdadera, el código se repite.

CUERPO
cin >> numero

Código que se ejecuta en cada iteración. Aquí debe haber algo que eventualmente cambie la condición, o el bucle nunca termina (bucle infinito ♾️).

// ejemplo práctico
// Suma números hasta que el usuario ingrese 0
int n, suma = 0;
cin >> n;

while (n != 0) {
    suma += n;
    cin >> n;
}
cout << "Suma total: " << suma;
// comparación

¿Cuándo usar cada uno?

Característica for while
Iteraciones Conocidas de antemano Desconocidas, dependen de condición
Contador Integrado en la sintaxis Debes manejarlo manualmente
Legibilidad Muy clara para rangos Clara para condiciones dinámicas
Ejemplo típico Recorrer un arreglo, tabla de multiplicar Leer entrada hasta que sea válida, menú
¿Son intercambiables? Sí — pero uno es más natural que el otro según el caso

La regla de oro

🔢

Usa FOR cuando...

  • Sabes cuántas veces repetir
  • Tienes un rango definido
  • Recorres un arreglo o lista
  • Usas un contador claro
❓

Usa WHILE cuando...

  • No sabes cuántas iteraciones habrá
  • Dependes del usuario o un dato externo
  • La condición es dinámica
  • Haces menús o validaciones

💡 Clave para recordar

Ambos hacen lo mismo, pero el código más legible es el que usa el bucle más natural para cada situación. Un buen programador no solo lo hace funcionar — lo hace entendible.
